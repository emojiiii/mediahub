# Security Policy

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability.

Use GitHub's private vulnerability reporting form:

https://github.com/emojiiii/mediahub/security/advisories/new

Include the affected version or commit, deployment profile, reproduction steps,
impact, and any suggested mitigation. Do not include real credentials, access
keys, session cookies, signing keys, or user data in the report.

Security fixes are developed privately and disclosed after a patched release is
available. The `master` branch is the only supported development line until the
project begins publishing versioned releases.
